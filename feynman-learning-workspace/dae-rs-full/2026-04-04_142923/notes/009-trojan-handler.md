## 口语化讲解区（外婆版）

**Trojan 协议的聪明之处**

普通代理协议（HTTP/SOCKS5）一眼就能被识别——网络审查者一看"这是代理流量"，直接封掉。

Trojan 的设计思路完全不同：**你先跟代理服务器建立 HTTPS 连接**（TLS握手），这时候在网络上看起来就是普通的浏览网页。然后在 TLS 加密层里面，"塞"进代理协议内容。

就像：快递柜表面上是一个普通的密码箱（TLS 加密），但里面实际上装着你要寄的东西（代理协议）。

**多后端容灾**

这个 handler 支持配置多个 Trojan 服务器后端（`backends`）。一个挂了，自动换另一个。按顺序轮着试（RoundRobin）。

**密码校验用了"常量时间比较"**

普通字符串比较：如果密码第一个字符就错了，函数直接返回 false，耗时短。攻击者可以通过测量响应时间猜出密码第一位是什么。

`ConstantTimeEq` 的做法是：无论对错都要把所有字符都比较完，耗时一样，攻击者无法通过时间差异窃取密码。

## 专业结构区（同行版）

### Trojan 协议帧格式（TLS 握手之后）

```
[密码 (56 bytes)][0x0D 0x0A]     ← 密码 + CRLF 结尾
[命令 (1 byte)][地址类型 (1 byte)][地址][端口 (2 bytes)][0x0D 0x0A]
```

### 命令类型

| 值 | 命令 | 含义 |
|----|------|------|
| 0x01 | Proxy | TCP 代理连接 |
| 0x02 | UdpAssociate | UDP 关联 |

### 地址类型

| 值 | 类型 | 格式 |
|----|------|------|
| 0x01 | IPv4 | `[4字节IP][2字节端口]` |
| 0x02 | 域名 | `[1字节长度][域名][2字节端口]` |
| 0x03 | IPv6 | `[16字节IP][2字节端口]` |

### 多后端轮询

```rust
fn next_backend(&self) -> &TrojanServerConfig {
    let idx = self.current_index.fetch_add(1, Relaxed) % self.backends.len();
    &self.backends[idx]
}
```

原子计数器递增，取模得到后端索引。第一个后端挂了，自动跳到下一个。

## 关键应用场景

**系统中做什么**：
Trojan handler 是 dae-rs 作为 Trojan 客户端的核心——连接到 Trojan 服务器，发送密码和目标地址，在 TLS 隧道内透传用户流量。

**没了会怎样**：
dae-rs 无法作为 Trojan 客户端运行，无法连接到 Trojan 服务器。

## 大白话总结（外婆版）

> Trojan handler 就是"伪装成网购的代购"。
>
> 你去代购网站下单，表面上是普通的 HTTPS 浏览流量，网络审查者以为是"这个人在正常网购"。但代购网站收到订单后，帮你真正去买你要的东西，然后寄给你。
>
> 关键设计：
> 1. **先 TLS 握手**：就像打开一个 HTTPS 网站，网络上谁都看不出来这是代理
> 2. **密码 56 字节**：固定的"会员卡号"，TLS 隧道建立后第一件事就验证这个
> 3. **多后端轮询**：有多个代购网站可用，一个挂了自动换另一个
> 4. **密码用常量时间比**：防止黑客通过响应时间猜出你的密码
